# MVP 路线图

## 1. MVP 目标

MVP 的目标是验证 MyGrad 是否能帮助学生完成基础选校信息查询和申请过程管理。

第一版不追求完整社区、复杂 AI 或精细后台，而是优先打通“填写背景 - 浏览项目 - 加入申请清单 - 管理 DDL 和任务”的闭环。

## 2. 阶段拆分

### 第 1 阶段：网站基础框架

- 初始化 Next.js、TypeScript、Tailwind CSS 和 shadcn/ui
- 建立基础路由、布局、导航和响应式页面结构
- 接入基础代码规范和部署配置

交付物：可运行的网站骨架、首页、登录页、工作台占位页面、基础 UI 组件体系。

### 第 2 阶段：用户系统与学生背景档案

- 接入 Supabase Auth
- 实现注册、登录、退出
- 实现学生背景档案表单与保存

交付物：用户可登录，并可填写和更新申请背景。

### 第 3 阶段：学校与专业数据库

- 建立学校和项目数据模型
- 实现学校列表、项目列表和详情页
- 支持基础筛选和搜索

交付物：用户可浏览学校与项目，项目详情包含官网、DDL、要求和来源信息。

### 第 4 阶段：规则版选校推荐

- 基于背景档案和项目要求建立简单规则
- 输出彩票、冲刺、匹配、稳妥四类推荐
- 允许用户将推荐项目加入申请清单

交付物：初版选校分层结果，推荐结果可转为申请目标。

### 第 5 阶段：申请清单、DDL 和任务管理

- 实现申请清单
- 实现任务创建、完成和截止日期提醒字段
- 在工作台展示近期 DDL 和待办

交付物：用户可管理多个申请项目，并查看关键任务和 DDL。

## 3. MVP 验收标准

- 新用户可以注册并登录
- 用户可以填写完整背景档案
- 用户可以浏览学校和项目详情
- 用户可以把项目加入申请清单
- 用户可以创建并完成申请任务
- 用户可以看到项目 DDL 和近期任务
- 规则版选校推荐能输出可解释的初步分类

## 4. 暂缓功能

- AI 文书生成与修改
- Mentor 付费咨询
- 完整社区评论互动
- Offer 复杂评分模型
- 后台复杂审核流
- 多语言版本

## 5. 近期优先级

1. 完成技术栈初始化和 UI 框架
2. 完成用户认证与学生档案
3. 完成学校和项目数据模型
4. 完成申请清单与任务管理
5. 完成规则版推荐闭环
