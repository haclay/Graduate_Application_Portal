# Codex 后续任务拆分

## 1. 使用原则

- 每次只执行一个明确阶段或一个小功能
- 开发前先检查当前文件结构和 Git 状态
- 不跨阶段提前开发功能
- 每次完成后运行必要检查、提交 commit 并 push
- 涉及数据库、认证或部署前先生成计划并确认

## 2. 第 1 阶段任务：网站基础框架

建议指令：

```text
请开始第 1 阶段：网站基础框架。请初始化 Next.js、TypeScript、Tailwind CSS、shadcn/ui，建立基础路由、布局、导航和首页/登录页/工作台占位页面。不要实现真实业务逻辑。完成后运行检查、commit 并 push。
```

验收标准：

- 本地项目可以启动
- 首页可以访问
- 基础布局和导航存在
- 没有真实业务功能代码

## 3. 第 2 阶段任务：用户系统与学生背景档案

建议指令：

```text
请开始第 2 阶段：用户系统与学生背景档案。请先规划 Supabase Auth 和 student_profiles 数据结构，再实现注册、登录、退出和学生背景档案表单。完成后运行检查、commit 并 push。
```

验收标准：

- 用户可以注册和登录
- 用户可以退出
- 用户可以填写和保存背景档案
- 用户只能访问自己的档案

## 4. 第 3 阶段任务：学校与专业数据库

建议指令：

```text
请开始第 3 阶段：学校与专业数据库。请根据 docs/03_DATABASE_DRAFT.md 建立学校和项目相关数据模型，并实现学校列表、项目列表和详情页。先使用少量种子数据，不要接入爬虫。
```

验收标准：

- 学校列表可访问
- 项目列表可访问
- 项目详情展示官网、DDL、申请要求和来源信息
- 支持基础筛选或搜索

## 5. 第 4 阶段任务：规则版选校推荐

建议指令：

```text
请开始第 4 阶段：规则版选校推荐。请基于学生背景档案和项目要求实现初版规则，将项目分为彩票、冲刺、匹配、稳妥，并展示推荐理由。不要接入 AI。
```

验收标准：

- 用户可看到分层推荐
- 每个推荐项目有简短理由
- 用户可将项目加入申请清单

## 6. 第 5 阶段任务：申请清单、DDL 和任务管理

建议指令：

```text
请开始第 5 阶段：申请清单、DDL 和任务管理。请实现申请清单、任务列表、任务状态、DDL 展示和工作台近期提醒。
```

验收标准：

- 用户可添加和管理申请项目
- 用户可创建、完成和查看任务
- 工作台展示近期 DDL

## 7. 后续开发注意事项

- 所有用户私有数据都需要权限隔离
- 学校和项目数据需要保留来源链接和更新时间
- AI 功能必须明确标注为辅助建议
- 不要在仓库中提交 `.env` 或密钥
- 每个阶段完成后更新 README 和相关 docs
